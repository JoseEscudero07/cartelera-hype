##  Organización del módulo videos

Se estructuró el módulo utilizando carpetas adicionales:

- `dto/`: para definir la forma de los datos de entrada y salida
- `entities/`: para representar la estructura del modelo de datos

## Justificación

- Separación clara de responsabilidades
- Mejora en la mantenibilidad
- Escalabilidad del código a futuro

## Fuente de datos

Se almacenó el mock de datos en un archivo local (`src/data/mock-youtube-api.json`) simulando una respuesta de una API externa.
